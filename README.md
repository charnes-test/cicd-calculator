# CI/CD пайплайн для Python-приложения

Этот проект демонстрирует реализацию полного цикла автоматизации разработки (CI/CD) для веб-сервиса на базе Flask. Основная цель проекта — показать настройку автоматического тестирования, сборки и доставки приложения в реестр контейнеров с использованием GitHub Actions.

## Описание работы

В репозитории настроен автоматический сценарий (workflow), который срабатывает при каждом обновлении ветки `main`. Процесс разделен на два последовательных этапа:

### 1. Непрерывная интеграция (Continuous Integration)
На этом этапе происходит проверка качества кода:
*   **Linting:** Запуск анализатора кода (`flake8`) для проверки соблюдения стандартов оформления.
*   **Unit Testing:** Выполнение автоматических тестов (`pytest`) для проверки корректности работы функций.
*   **Quality Gate:** Если на этом этапе возникают ошибки, процесс сборки принудительно останавливается, что предотвращает попадание нерабочего кода на следующие этапы.

### 2. Непрерывная доставка (Continuous Delivery)
Этот этап запускается только при условии успешного прохождения всех тестов:
*   **Build:** Автоматическая сборка оптимизированного Docker-образа приложения.
*   **Push:** Авторизация в Docker Hub с использованием зашифрованных переменных (GitHub Secrets) и отправка готового образа в публичный реестр.

## Стек технологий

*   **Приложение:** Python 3.9, Flask
*   **Тестирование:** Pytest, Flake8
*   **Контейнеризация:** Docker
*   **Автоматизация:** GitHub Actions
*   **Реестр образов:** Docker Hub

## Запуск из Docker Hub

Благодаря настроенному процессу CD, актуальная версия приложения всегда доступна в реестре. Для запуска не требуется клонировать репозиторий или устанавливать зависимости Python.

Выполните команду:

```bash
docker run -p 5000:5000 charnes/cicd-calculator:latest
